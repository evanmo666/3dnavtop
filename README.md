# 3D设计师导航网站 (3dnav.top)

一个专为3D设计师打造的资源导航网站，基于Next.js 15 + TypeScript构建。

## 🚀 项目特色

- **现代化技术栈**: Next.js 15 + TypeScript + Tailwind CSS
- **响应式设计**: 完美适配桌面端和移动端
- **智能搜索**: 支持实时搜索和分类筛选
- **后台管理**: 完整的链接管理系统
- **文件存储**: 真实文件操作，数据永久保存

## 📊 项目数据

- **总链接数**: 61个精选资源
- **特色链接**: 15个重点推荐
- **分类数量**: 9个主要分类
- **管理模式**: 文件模式 - 数据永久保存到文件系统

## 🛠️ 技术架构

### 前端技术
- **框架**: Next.js 15 (App Router)
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **动画**: Framer Motion
- **图标**: Heroicons
- **广告系统**: Google AdSense (自动广告)

### 后台管理
- **数据存储**: 基于文件系统的真实数据操作
- **API路由**: Next.js API Routes
- **文件操作**: 自定义文件操作工具
- **权限控制**: 简化的管理界面
- **数据备份**: 自动备份机制

### 数据管理
- **主数据文件**: `src/app/data/links.ts` (真实数据文件)
- **文件操作**: 直接修改TypeScript数据文件
- **备份系统**: 每次修改前自动创建备份
- **API端点**: 
  - `GET /api/links` - 获取所有链接
  - `POST /api/links` - 添加新链接 (永久保存)
  - `GET /api/links/[id]` - 获取单个链接
  - `PUT /api/links/[id]` - 更新链接 (永久保存)
  - `DELETE /api/links/[id]` - 删除链接 (永久保存)

## 🚀 快速开始

### 环境要求
- Node.js 18+
- npm 或 yarn

### 安装步骤

1. **克隆项目**
```bash
git clone <repository-url>
cd 3dnav
```

2. **安装依赖**
```bash
npm install
```

3. **启动开发服务器**
```bash
npm run dev
```

4. **访问应用**
- 前台: http://localhost:3000
- 后台管理: http://localhost:3000/admin/links

### 构建部署

```bash
# 构建生产版本
npm run build

# 启动生产服务器
npm start
```

## 📝 使用说明

### 文件模式说明

当前项目运行在**文件模式**下：
- ✅ 所有功能正常工作
- ✅ 数据永久保存到文件系统
- ✅ 每次修改前自动创建备份
- ✅ 支持完整的增删改查操作
- ⚠️ 需要文件系统写入权限

### 添加新链接

1. 访问 `/admin/links/new`
2. 填写链接信息：
   - 标题 (必填)
   - URL (必填)
   - 描述 (可选)
   - 分类 (必填)
   - 子分类 (可选)
   - 是否特色
   - 排序权重

3. 点击"添加链接"保存 (永久保存到文件)

### 编辑链接

1. 在链接管理页面点击"编辑"
2. 修改链接信息
3. 点击"更新链接"保存 (永久保存到文件)
4. 或点击"删除链接"删除 (永久从文件删除)

### 数据文件结构

```typescript
interface ILink {
  _id: string;           // 唯一ID
  title: string;         // 链接标题
  url: string;          // 链接URL
  description?: string;  // 描述信息
  category: string;     // 主分类
  subcategory?: string; // 子分类
  featured?: boolean;   // 是否特色
  order?: number;       // 排序权重
  createdAt: Date;      // 创建时间
  updatedAt: Date;      // 更新时间
}
```

### 备份和恢复

系统会在每次修改前自动创建备份文件 `src/app/data/links.backup.ts`。如果需要恢复：

```bash
# 手动恢复备份
cp src/app/data/links.backup.ts src/app/data/links.ts
```
